## Context

本系统目标:不停增量获取全球信息,从中找到对世界有影响的信息,推导影响逻辑,发现投资机会。

本设计要解决的根本张力:**结构化流程 vs 大模型的专家直觉**。过度结构化会把大模型降维成传统量化系统,阉割掉它唯一不可替代的能力 —— 跨域模式识别与战术顿悟(我们这次对话里"厄尔尼诺→地下管网排水""内存→接口芯片二阶"这类判断,都来自直觉而非流程)。

既有输入:
- 研究总结(2026-03-12):排除性标准、可交易五要素(时间锚点/量化/范围/可验证/因果链清晰)、"交易要确定性不是可能性"。
- 近期实战对话沉淀:7 类蛛丝马迹、"好公司 ≠ 好买点"(誉帆教训)、last30days 只能当注意力层。

## Goals / Non-Goals

**Goals:**
- 保护并最大化大模型的专家直觉与跨域顿悟,让**生成环节自由**。
- 结构只放在两端:**输入**(喂干净的料)与**反馈**(战绩闭环)。
- 给大模型补上它从文本训练里缺失的、棋手/球员都有的**市场反馈闭环**。
- 把契约定位成"记忆 + 反馈底座",而非"流程笼子"。

**Non-Goals:**
- 不规定模型"怎么推理";不做重型量化打分流程。
- 不追求"不漏";明确容忍漏、优先压制错。
- 本期只定义契约,不实现采集源 / 漏斗 / 回测代码。

## Decisions

**D1 直觉优先,结构外移。** 生成环节自由放开;结构放在输入(`signal-contract` 喂好料)和反馈(`thesis-contract` 战绩记录)两端。
- 理由:Kahneman-Klein《Conditions for Intuitive Expertise》—— 直觉可信需两个条件:环境规律 + 带反馈的长期练习。市场是低效度环境,且大模型的市场直觉来自文本(未经 P&L 校准),所以它"感觉像大师、校准像业余"。唯一解药是**反馈闭环**,不是更多流程规则。
- 备选(已否决):重型 8 字段推导规范 —— 会阉割直觉,退回传统量化系统。

**D2 `thesis-contract` 翻转。** 从"必填字段表格"改为"**自由论点 + 1 个对抗证伪 + 1 条战绩记录 + 可追溯 + 置信标注**"。契约不规定推理结构,只规定"必须能被踢馆、必须能被复盘"。

**D3 `logic_score` 与 `buy_point` 强制分离。** 这是少数必须硬的结构,因为它防的是一类**确定的认知错误**(誉帆:把好公司当成好买点),不是限制直觉。

**D4 错 / 漏不对称,默认"宁可漏不可错"。** 沿用研究总结立场:容忍错过(漏),优先避免做错(错)。

**D5 `last30days` = 注意力 / 情绪传感器,非主信号源。** 它满足 `signal-contract` 的来源字段,但不享有更高权重。

**D6 契约 = 记忆 / 反馈底座。** 契约的职责是记下"模型想了什么、后来发生了什么",让直觉能拿自己的战绩做模式匹配,从而越用越准。

**D7 schema / 语义分工。** JSON Schema 是结构层唯一事实源,负责 required 字段、枚举、内部对象结构;Python 校验器只负责 schema 表达不了的语义不变量,例如对抗证伪独立性、近重复、confirmed thesis 支撑、买点纪律。

**D8 反馈边界。** 本 change 只实现战绩 outcome 回填,并显式保留 `calibrate()` 的 not-implemented 边界。完整校准统计、命中率分桶、置信度校准偏差、模式反馈写回另开 `feedback-calibration` change。

**D9 反向进料是独立入口。** 市场异动/反向进料不是普通新闻源,而是 `market_move -> backtrace_news -> signal(signal_origin=market_move)` 的独立入口。它盘后扫描,可触发事件级分析,但不是实时交易引擎。本 change 不拆 `market-event-contract`,只在 `signal-contract` 中锁 `signal_origin`。

**D10 分析层三段式。** ③ 分析层是 `自由生成 -> 完整性批判 -> 对抗证伪`。完整性批判只问遗漏的二阶影响,产出候选追加 thesis,不得强加推理模板。

**D11 全球到 A股传导显式化。** 跨市场推导由 `thesis-contract` 的 `origin_market`、`target_market`、`transmission_path` 承载,并由 `target-contract.target_market` 承接。传导路径通过两步抽取产生,不得要求自由 body 生成时填字段。

**D12 人的裁定与市场 outcome 分离。** `human_decisions` 独立于 `track_record`:前者记录用户 accepted/rejected/overridden 与理由,后者记录市场结果。两者都写入记忆底座,但只有成熟市场结果可进入 calibration signal。

**D13 空仓是合法输出。** 周级输出增量更新 touched themes 并做一次完整性批判扫描;当无够格机会时,输出"无推荐 + 原因"而不是硬产名单。

## Risks / Trade-offs

- 完全放开生成 → 退回 last30days 第一轮的喊单式 slop。 → **缓解:对抗证伪 + 强制战绩记录托底**,信任生成但怀疑自信。
- 反馈闭环需时间积累才有用,冷启动期直觉未校准。 → **缓解:本 change 先完成 outcome 回填边界;冷启动靠对抗证伪 + 历史基率;接受早期低置信、轻仓。**
- 漏(错过机会)无法靠流程消灭。 → **缓解:反向进料(盯价格异动倒查新闻)+ 完整性批判("还有哪个二阶影响没想到");但接受漏不可消灭。**
- 对抗证伪方也是 LLM,可能与生成方同源偏差。 → **缓解:强制 instance/run 不同 + persona 不同;必要时换不同模型,高 stakes 引入人工复核。**
- 去重 / 排除规则可能误杀真信号。 → **缓解:排除只做"明显不可交易",从轻;保留原始载荷可重跑。**
- 每日反馈可能把短期噪声污染周级判断。 → **缓解:区分 `outcome_raw` 与 `calibration_signal`,通过成熟度过滤后再喂回周级。**
- 事件驱动可能退化为追热点。 → **缓解:事件 hard gate + 触发预算 + `trigger_reason` 记录。**

## Open Questions

- 完整校准统计的具体分桶和回写方式由后续 `feedback-calibration` change 定义。
- 战绩记录的"结果"自动回填数据源待定(价格 API + 时间窗对齐)。
- 反向进料(异动倒查)盘后扫描的数据源待定;完整 `market-event-contract` 留给后续 change。
