## Why

每天涌入的全球信息有数百条,绝大多数"看起来有价值"却不可交易。当前的分析靠记忆和临场判断,人与 AI 的口径每一次都在漂移:信号、逻辑、买点被混为一谈,priced-in(是否已被定价)和证伪经常被遗忘。研究总结(2026-03-12)已经确立"交易要的是确定性,不是可能性",并主张用"排除性标准"而非"筛选标准"——但这些认知还散落在文档和对话里,没有变成人和 AI 都漂不了的稳定参照物。

本变更把这套分析的参照物**外化成三份数据契约**,作为整个 news→opportunity 流水线的地基,先于任何采集与管道代码。契约的目的不是冻结需求,而是让上千次增量分析共享同一套不漂移的 schema,并通过显式、有版本的契约演进来取代隐式的上下文漂移。

## What Changes

- 新增三份核心数据契约(spec),作为流水线的稳定参照物:
  - `signal-contract`:定义"一条信号"的结构——来源+日期出处、信号类型(供需瓶颈 / 政策 / 天气气候 / 出口管制地缘 / 技术拐点)、传导机制、量化量级与范围、可交易性维度(时间锚点 / 量化指标 / 范围明确 / 可验证性 / 因果链清晰度),以及排除标志(研究总结的 5 大致命缺陷:时间模糊、内容模糊、影响模糊、高冗余、低信噪比)。
  - `thesis-contract`:定义一条"因果论点"的结构——证据(每条带来源+日期)、支撑逻辑 / 机制、方向(利多 / 利空)与受影响行业、候选标的及入选理由、证伪信号、对冲变量、跨信号共振。
  - `target-contract`:定义一个"可投标的"的结构——**强制分离 `logic_score`(逻辑分)与 `buy_point`(买点 / 估值分)**、催化剂日历、证伪触发条件、新鲜度 / 已涨幅(priced-in),以及观察名单状态(watch / buy-zone / hold / exit)。
- 确立"契约先行"工作流:后续的采集源、LLM 分析 prompt、watchlist 存储都必须满足这三份契约,而不是各写各的口径。
- 把两份既有认知固化进 schema:研究总结(2026-03-12)的可交易必备要素与排除标准;近期实战对话沉淀的"7 类蛛丝马迹"(产能售罄 / 交期拉长 / 长协涨价 / 大客户锁单 / 出口管制 / 首付款落地 / 资本开支大增)与"好公司 ≠ 好买点"纪律。

## Capabilities

### New Capabilities

- `signal-contract`:一条原始信号的 schema 与不变量。明确什么是信号、它的出处与时效、属于哪类传导、以及它是否可交易(可交易性维度打分 + 5 大排除标志)。是排除性漏斗的入口契约。
- `thesis-contract`:一条因果论点的 schema 与不变量。把一条或多条信号转成"信息 → 支撑逻辑 → 方向 / 行业 → 候选标的 → 证伪 / 对冲"的固定结构,强制每条论点都带证伪项,杜绝只讲利多不讲反方。
- `target-contract`:一个可投标的的 schema 与不变量。强制把"逻辑分"与"买点分"做成两个独立字段,挂催化剂日历与证伪触发条件,落"观察名单 + 触发条件"的实战纪律。

### Modified Capabilities

（无——全新项目,这是首批 spec。）

## Impact

- 建立系统数据模型的单一事实来源;所有下游层(采集 / 四级排除漏斗 / LLM 因果分析 / 时效定价 / watchlist / 复盘校准)都按这三份契约对接。
- 本变更不涉及实现代码,只定义契约。
- `last30days` 在本系统中被定位为"注意力 / 情绪传感器"输入,必须满足 `signal-contract` 的来源字段与排除标志,而非作为主信号源。
- 后续变更(采集源接入、四级排除漏斗、信号源校准与回测)均以这三份契约为前置依赖。
