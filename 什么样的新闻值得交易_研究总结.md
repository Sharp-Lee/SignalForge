# 什么样的新闻值得交易？研究总结

**研究日期**：2026-03-12
**分析团队**：信息论专家、市场微观结构专家、行为金融专家、实战交易员

---

## 核心问题

现有新闻筛选方案的困境：
1. **假阳性率过高**：任何筛选框架都让大部分新闻"看起来有价值"
2. **无法执行**：一天几百条新闻筛选出几百只股票 = 没有筛选
3. **成本过高**：对每条新闻深度分析token成本极高
4. **分类陷阱**：按事件类型分类会预设价值，但同类新闻价值差异巨大

---

## 案例分析：水污染防治新闻

### 新闻内容
> 生态环境部部长黄润秋3月12日在十四届全国人大四次会议第三场"部长通道"上介绍，将出台深化水污染防治有关文件

### 四位专家一致结论：**不值得交易**

#### 1. 信息论专家（3/10分）
- **惊讶度**：2/10（P(E)=0.4-0.6，常规事件）
- **信息增益**：仅20%（缺时间、力度、范围）
- **信噪比**：0.3-0.5（远低于1.5阈值）
- **冗余度**：70-80%（市场已预期）
- **关键洞察**：这是"意向信号"，距离可执行信息差2-3个层级

#### 2. 市场微观结构专家
- **因果链清晰度**：低（5个不确定环节）
- **验证时滞**：3-6个月
- **流动性风险**：中-高（环保股多为中小市值）
- **共识分散度**：高（散户vs机构认知差异大）
- **交易窗口**：2-4周，但有效时间仅3-5天

#### 3. 行为金融专家（C级）
- **投资者注意力**：中等偏低，缺乏意外性
- **市场反应**：初期反应不足，难以持续
- **羊群效应**：短期跟风，持续性差
- **叙事驱动**：与当前主流叙事契合度弱

#### 4. 实战交易员（不交易）
- **标的选择**：无法判断具体方向，缺乏依据
- **入场时机**：观望为主
- **风险收益比**：1:1.5（不理想）
- **机会成本**：资金应留给确定性更高的机会
- **核心原则**：**交易要确定性，不是"可能性"**

## 关键发现：不值得交易的新闻特征

### 致命缺陷（5个维度）

1. **时间模糊**
   - "将出台"、"计划"、"研究"、"拟"
   - 无法确定交易窗口
   - 资金占用时间不可控

2. **内容模糊**
   - "有关文件"、"相关政策"、"涉及领域"
   - 无法映射到具体标的
   - 无法评估影响力度

3. **影响模糊**
   - 缺乏量化指标（金额、百分比、数量）
   - 受益范围不明确
   - 因果链断裂（5+个不确定环节）

4. **高冗余**
   - 市场已有预期（常规议题）
   - 重复报道
   - 季节性事件（两会、年报）

5. **低信噪比**
   - SNR < 1.0（信号被噪声淹没）
   - 无法产生超过市场波动的价格影响
   - 惊讶度低（P(E) > 0.4）

## 可交易新闻应该是什么样？

### 信息论专家给出的对比示例

**不可交易**（当前案例）：
> "生态环境部将出台深化水污染防治有关文件"

**可交易**（理想形态）：
> "生态环境部将于Q2出台水污染防治新规，COD排放标准提升30%，涉及长江流域化工企业，预计带动300亿治理投资"

### 可交易新闻的必备要素

✓ **时间锚点**：Q2（季度级精度）
✓ **量化指标**：30%、300亿
✓ **明确范围**：长江流域化工企业
✓ **可验证性**：排放标准提升（可监测）
✓ **因果链清晰**：标准提升→企业必须投资→设备商受益

## 解决方案：排除性标准而非筛选标准

### 第一阶段：低成本快速排除（规则过滤）

**立即排除的新闻特征**：

1. **时间模糊词**：将、拟、计划、研究、考虑
2. **内容模糊词**：有关、相关、涉及、等
3. **无量化指标**：没有数字、百分比、金额
4. **常规性事件**：两会、例会、年度报告、季度会议
5. **重复新闻**：Jaccard相似度 > 0.25

**排除率目标**：90%+

### 第二阶段：轻量级模型初筛（Haiku/传统ML）

对通过第一阶段的新闻：
- 评估信息完整度（时间+内容+影响）
- 计算惊讶度估计值
- 检测市场预期程度

**排除率目标**：80%

### 第三阶段：深度分析（Opus）

对通过前两阶段的新闻（约2%）：
- 完整的信息论分析
- 市场微观结构评估
- 交易可行性判断

### 第四阶段：相对排序 + Top N

- 不问"是否有价值"，而问"相对价值排序"
- 每天只交易排名前5-10的机会
- 动态阈值：新闻质量差的日子可以不交易

---

## 反向思路：从市场反应倒推

### 方案：监控市场异常 → 寻找新闻

**优点**：
- 只分析"市场已经在反应"的新闻
- 大幅减少需要分析的新闻量
- 避免"有信息但市场不care"的新闻

**实施**：
1. 监控异常波动（价格、成交量、订单流）
2. 当股票出现异常时，反查新闻
3. 只对有市场反应的新闻进行深度分析

**缺点**：
- 可能错过市场反应慢的机会
- 需要实时市场数据

## 实战原则

### 核心理念
**交易要的是确定性，不是"可能性"**

### 操作准则
1. **知道了但不做**：大部分新闻属于这一类
2. **保持现金**：等待更好的机会
3. **观察名单**：将潜在标的加入自选，等待触发条件
4. **重新评估触发条件**：
   - 政策正式发布，明确细节
   - 龙头股放量突破
   - 公司公告重大订单
   - 板块指数突破前期高点

## 下一步研究方向

1. **建立排除规则库**：收集更多"不值得交易"的新闻特征
2. **量化评分系统**：开发快速评分模型（0-100分）
3. **历史案例库**：回测不同类型新闻的实际交易价值
4. **成本优化**：设计多级漏斗的具体实现方案
